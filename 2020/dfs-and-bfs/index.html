【数据结构与算法】深度优先搜索和广度优先搜索

Posted on 2020-02-06 In 数据结构与算法 Views: Valine:
Word count: 11k Time cost: 19 mins.

前言

深度优先搜索（DFS）和广度优先搜索（BFS）是两种经典的遍历（搜索）树或图中节点的算法。以树为例，DFS 的思路是沿着子树尽可能深地搜索树的分支，到达叶子结点后通过回溯重复上述过程，直到所有的结点都被访问。BFS 的思路则是一层一层地访问结点，直到完成遍历。

由于 DFS 和 BFS 的这种差异， BFS 一般用来求解最短问题，而 DFS 书写起来比较简单，一般因此对于不是最短问题的情况，我们优先考虑。然而事事无绝对，DFS 也是可以解决最短问题，BFS 也是一样。很多情况下，两者可以交替使用，比如本章要将的岛屿问题。

不管是 DFS 还是 BFS，本质上都是搜索，而这种搜索通常来说都是暴力的搜索。因此当你需要对问题的所有可能情况进行穷举的时候，你就应该想到 DFS 和 BFS。而第 16 章要讲解的回溯法，也是 DFS 的一种，也是一种暴力搜索方法，只不过回溯法会涉及到进入和回溯过程。

本章会通过 LeetCode 中的路径和问题和岛屿问题来详细讲解 DFS 和 BFS 的思路。

LeetCode 中的路径和系列是典型的可以用 DFS 来解决的题目。

返回 true, 因为存在目标和为 22 的根结点到叶子结点的路径 5->4->11->2

解法一 - DFS 递归

思路

题目要找到一条根结点到叶子结点的路径，并且路径上所有结点的值之和等于给定的目标值。也就是说，需要搜索这棵二叉树的不同路径（根结点到叶子结点）。如果找到一条符合的路径，返回 true；如果所有的路径都不符合要求，则返回 false。因为题目要求我们只需要找到符号要求的一条路径，所以使用 DFS 来处理本题就显而易见了。

通常来讲，DFS 有递归和迭代两种实现方式。因为树结构天然具有递归的特性（子树性质和整棵树性质一致），使用递归可以很容易将整棵树问题转换成子树问题。当我们层层递归到最小的子树时，这棵最小子树的解往往能够很容易得到（也被称为递归出口），然后再一步步回溯就能得到原问题的解。

小提示：树的题目，优先考虑使用 DFS 递归解决

当我们处理递归问题的时候，如何定义递归出口是非常重要的一步（递归出口指的是递归函数可以直接处理的最简单子问题）。对于本题，递归出口是当前子树只有一个结点时（该结点是整棵树的叶子结点），此时需要在递归出口判断当前路径是否符合要求。

LC 124. 二叉树中的最大路径和

题目描述

给定一个非空二叉树，返回其最大路径和。

本题中，路径被定义为一条从树中任意结点出发，达到任意结点的序列。该路径至少包含一个结点，且不一定经过根结点。

示例：
输入：[1,2,3]

输出：6

思路

我们知道，树本身是图的一种特例，找树中任意起点的最大路径和其实就是找图中的最大权值和。

我们先思考一种简单情况 —— 只有三个结点的子树：

它包含的路径和总共有 6 种：

  1. $b$
  2. $c$
  3. $a$
  4. $a+b$
  5. $a+c$
  6. $b+a+c$

假设上面这棵子树是整棵树的一部分，那么对于获取最终答案而言，该子树可能的贡献有如下三种情况：

  1. 整棵树的最大路径和就在这棵子树当中，不再经过这棵子树外的其他结点，则最终结果是 $max(b, a, c, b+a+c)$。
    • $b$ 和 $c$ 分别是单结点子树的最大值（也就是该结点本身），表示最大路径和在子树 $a$ 的子树当中。
    • 取 $a$ 表示，$b$ 和 $c$ 都无法给最大路径和做贡献，也就是都小于 0。
    • 取 $a+b+c$ 表示 $b$ 和 $c$ 都给最大路径和做贡献，也就是都大于 0。
  2. 最大路径和包含结点 a ，并且通过 a 扩展到子树外的其他结点。 这里 $parent(a)$ 表示最大路径和中由结点 a 往外延伸的剩余部分路径和，则最终结果是 $max(b,c) + a + parent(a)$。
  3. 不包含子树 a


（图 5.1.3.1）

如果把结点 b 和 c 抽象成任意一棵子树的左右子树，我们便可以将子树 a 的情况扩展到整棵树，实现特例到全局的抽象。遇到这类可以用同种逻辑处理问题及其子问题的情况时，使用递归思想是一个不错的思路。

设定一个全局变量存储最大路径和，不断递归搜索每棵子树更新该变量。

  • 首先搜索的是最左边的最小子树，不断回溯到更大一点的子树，直到 root 结点的整个左子树被搜索完成
  • 然后对 root 结点的右子树做同样的操作
  • 最后再判断包含 root 结点的情况

岛屿问题

LeetCode 中有几道题目都带着“岛屿”字眼，并且解题思路相似，因此我们简称为岛屿问题，这类题目同样是典型的可以用 DFS 和 BFS 来解决的题目。从数学层面来讲，本小节的题目都可以转化为无向图的连通分量问题，也就是等价类问题。连通分量简单的可以理解为该分量内部任意点之间都是可以相互连通的（如果想要详细了解等价类问题，可以自行搜索）。

LC 200. 岛屿数量

题目描述

给定一个由 ‘1’（陆地）和 ‘0’（水）组成的的二维网格，计算岛屿的数量。一个岛被水包围，并且它是通过水平方向或垂直方向上相邻的陆地连接而成的。你可以假设网格的四个边均被水包围。

示例：
输入：
11110
11010
11000
00000

输出：1

分析

题目要求计算岛屿的数量。岛的定义告诉我们岛的内部点与点之间都是相互连通的。因此，一个岛其实就是一个由点构成的无向连通子图。而求岛的数量，就是求整个无向图（这里将二维网络当成一个大的无向图）中的连通分量个数。当我们遇到图论中求连通分量的个数时，基本都可以使用DFS 、BFS 或者 并查集 (Union-Find) 方法来解决。下面就让我们来慢慢熟悉这一类题目的套路。

解法一 - DFS 递归

思路

线性扫描整个二维网络，找到第一个1，然后以该结点为开始结点来启动深度优先搜索。在深度优先搜索的过程中，将每个访问过的结点标记为 0，表示该结点已经被访问过了。那么，在整个线性扫描过程当中启动深度优先方法的次数就是岛屿的数量。
深度优先搜索的思路是沿着一个方向一直走，直到走到尽头，然后再尝试往其他方向搜索。对于本题而言，有四个方向-上、下、左、右。如果被访问结点的各个方向都搜索过了，则回溯到上一个结点。

解法三 - BFS 迭代

思路

同样地，我们可以使用广度优先搜索来解决该题。算法思路如下：

线性扫描整个二维网络，如果一个结点中包含 1 ， 则以该结点为开始结点启动广度优先搜索。而启动广度优先搜索的次数就是岛屿的数量。

  • 广度优先搜索通常需要辅助的队列。首先，将开始结点放入该队列中，然后迭代搜索队列中的每个结点。在迭代过程中，将当前结点符合要求的子结点放入队列中，访问完当前结点后，将该结点从队列中抛出。整个迭代过程直到队列为空时结束。这里同样通过将1设为0来标记已访问过该结点。

LC 305. 岛屿数量 II

题目描述

给定一个有 m 行和 n 列 的二维网络， 该网络最开始全部被水充满， 我们可以通过 addLand 来将一个位置的水变为陆地。 给定一组坐标来执行 addLand 操作，输出每次操作完后岛屿的数量。一个岛被水包围，并且它是通过水平方向或垂直方向上相邻的陆地连接而成的。你可以假设网格的四个边均被水包围。

示例：

输入：
m = 3, n = 3, positions = [[0,0], [0,1], [1,2], [2,1]]

输出：
[1,1,2,3]

思路

本题明显是 No.200 的变种，我们需要动态地求出每次 addLand 操作之后的无向图中的连通分量。而求连通分量的数量的问题都可以通过 DFS 、 BFS 或并查集来解决。
首先来看 DFS 和 BFS，任何通过 DFS 和 BFS 来解决的图类问题都有一个前提-图是被预先处理好的。而本题中的图是在动态变化的。因此这时候用 DFS 或 BFS 来处理本题就显得不那么高效了。

那么并查集呢？
经过路径压缩和按秩合并的 union 操作只需要常数时间。而 addLand 操作其实等价于 union 操作。那么高效的解法就显而易见了。

因此，如果需要求动态变化中的图的连通分量数量，使用并查集效率更好。

  • 空间复杂度：$O(M * N)$，这是并查集数据结构需要的空间。

总结

回顾整章，DFS 和 BFS 都属于树/图的搜索算法，两者在用于具体问题时各有优劣，比如：

  • 求给定图中两点之间最短路径或者检验图的二分性，使用 BFS 更优
  • 求图的拓扑排序，使用 DFS 更优
  • 求无向图的连通分量数量，两者差不多

两者在实现过程中使用的基础数据结构也有区别。在实际做题当中，一般使用栈来实现 DFS，队列来实现 BFS。

另外，关于 DFS 和回溯算法之间的关系：
这两者之间的关系界线是模糊的，网上的说法各不一样，在这里我们没必要过于纠结精确的定义。

对于 DFS，另外一个知识点也是值得注意的。在二叉树当中，DFS 可以被分为先序遍历、中序遍历和后序遍历，并且引申出一系列相关题目。

最后，本章的路径和问题和岛屿问题只详细讲述了两种算法的基本写法，而在实际的刷题过程中，我们可能会使用这两种基本写法的变种或延伸，比如运用双向搜索技巧、dijkstra 算法、A* 算法等等。而对这些技巧或变种算法的理解和应用，就有待读者在刷题中自行总结了。我们在刷题之后，需要花一些时间看看这些算法在工业界的应用，将理论与工业实践结合起来。比如，可以深入研究下 BFS 是如何运用在 cheney 算法中，如何用来跟踪垃圾回收（garbage collection）的。
